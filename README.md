# MAX ⇄ Telegram Bridge

**Зеркало мессенджера [MAX](https://max.ru) в Telegram.** Читай все свои чаты MAX
прямо в Telegram-группе и отвечай оттуда — от своего имени, не открывая MAX.

Мост заходит в MAX по **твоему личному аккаунту** (userbot), поэтому видит сразу
все диалоги. Каждый чат MAX становится отдельной **темой (веткой)** в
Telegram-супергруппе-форуме; ветки создаются автоматически по мере появления чатов.

```
   MAX чат A  ─┐                              ┌─ Telegram тема «Чат A»
   MAX чат B  ─┼──  [ MAX ⇄ TG Bridge ]  ─────┼─ Telegram тема «Чат B»
   MAX чат C  ─┘     (твой аккаунт MAX)        └─ Telegram тема «Чат C»
```

---

## ✨ Возможности

- 📥 **MAX → Telegram** — входящие сообщения зеркалятся в соответствующую тему:
  текст, **фото, видео, файлы, голосовые, аудио, стикеры**.
- 📤 **Telegram → MAX** — всё, что пишешь в теме, уходит обратно в тот же чат MAX
  **от твоего имени** (текст и вложения).
- 🧵 **Авто-ветки** — на каждый чат MAX создаётся отдельная тема; маппинг хранится
  в SQLite и переживает перезапуск.
- 🔄 **Синхронизация чатов** — при старте (и по команде `/sync`) создаёт ветки для
  всех уже существующих диалогов.
- 📞 **Уведомления о звонках** — пропущенные и завершённые звонки прилетают в ветку.
- ↪️ **Пересланные сообщения** — разворачиваются и пересылаются с пометкой об источнике.
- 🎞 **Стикеры и гифки** — стикеры конвертируются в картинку (с обрезкой фона),
  видеостикеры — в кадр, гифки уходят видео.
- ⚙️ **systemd-сервис** — работает в фоне с автозапуском и авто-перезапуском.

## 🚀 Установка в одну команду

На сервере **Ubuntu** (22.04/24.04) от root:

```bash
sudo bash <(curl -fsSL https://raw.githubusercontent.com/YOURGITHUB/max-tg-bridge/main/install.sh)
```

Скрипт сам поставит зависимости (Python, `ffmpeg`), скачает проект в
`/opt/max-tg-bridge`, спросит токен/ID группы/телефон, проведёт авторизацию в MAX
(введёшь код из SMS) и запустит сервис с автозапуском.

> Что подготовить заранее — см. [«Перед установкой»](#-перед-установкой).
> Удаление: `sudo bash /opt/max-tg-bridge/uninstall.sh`.

## 📋 Перед установкой

Понадобятся **три вещи**:

### 1. Telegram-бот и группа
1. [@BotFather](https://t.me/BotFather) → `/newbot` → получи **токен** (`TG_BOT_TOKEN`).
2. Отключи приватность: `/setprivacy` → выбери бота → **Disable**.
3. Создай **супергруппу** и включи в ней **Темы (Topics)**: *Управление группой → Темы*.
4. Добавь бота в группу **администратором** с правом **«Управление темами»**.
5. Узнай **ID группы** (`TG_GROUP_ID`, вида `-100…`): добавь в группу
   [@getidsbot](https://t.me/getidsbot), либо запусти `get_group_id.py` из проекта.

### 2. Аккаунт MAX
Только **номер телефона** твоего аккаунта MAX (`MAX_PHONE`, формат `+7…`).
Код из SMS введёшь при авторизации.

## 🛠 Установка вручную

Если не хочешь установщик — всё то же по шагам:

```bash
# зависимости
sudo apt update && sudo apt install -y python3 python3-venv python3-pip git ffmpeg

# проект
sudo git clone https://github.com/YOURGITHUB/max-tg-bridge.git /opt/max-tg-bridge
cd /opt/max-tg-bridge
python3 -m venv venv
./venv/bin/pip install -r requirements.txt

# настройка
cp .env.example .env
nano .env                       # TG_BOT_TOKEN, TG_GROUP_ID, MAX_PHONE

# авторизация в MAX (один раз, введи код из SMS; на сервере — внутри tmux)
./venv/bin/python login.py

# проверка
./venv/bin/python bot.py
```

Автозапуск через systemd:
```bash
sudo cp systemd/max-tg-bridge.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now max-tg-bridge
journalctl -u max-tg-bridge -f
```

## ⚙️ Настройка (`.env`)

| Переменная            | По умолчанию    | Описание |
|-----------------------|-----------------|----------|
| `TG_BOT_TOKEN`        | —               | Токен Telegram-бота от @BotFather |
| `TG_GROUP_ID`         | —               | ID супергруппы-форума (`-100…`) |
| `MAX_PHONE`           | —               | Телефон твоего аккаунта MAX (`+7…`) |
| `MAX_WORK_DIR`        | `max_session`   | Каталог файла сессии MAX |
| `MAX_SESSION`         | `session.db`    | Имя файла сессии MAX |
| `FORWARD_TG_TO_MAX`   | `true`          | Пересылать из Telegram обратно в MAX |
| `SHOW_SENDER_NAME`    | `true`          | Подписывать сообщения в темах именем отправителя |
| `SYNC_CHATS_ON_START` | `true`          | Создавать ветки для всех чатов при старте |
| `NOTIFY_CALLS`        | `true`          | Уведомления о звонках |
| `DB_PATH`             | `bridge.db`     | Файл базы со связями тем |
| `LOG_LEVEL`           | `INFO`          | Уровень логов (`DEBUG` для отладки) |

Команды в Telegram: **`/id`** — показать chat_id и id ветки, **`/sync`** —
синхронизировать список чатов.

## ⚠️ Важно понимать

- **Это userbot на неофициальном протоколе MAX.** Используется библиотека
  [`maxapi-python`](https://pypi.org/project/maxapi-python/). Это против правил
  сервиса MAX, и **теоретически возможна блокировка аккаунта**. Ты действуешь на
  свой риск, со своим аккаунтом.
- **Первый вход интерактивный** — нужно один раз ввести код из SMS (и пароль 2FA,
  если включён). На сервере запускай `login.py` внутри `tmux`/`screen`.
- **Свои сообщения не дублируются** в Telegram (анти-эхо), чтобы не было петли.
- **Скачивание файлов из Telegram ограничено 20 МБ** (лимит Telegram Bot API).
- **Голосовые** уходят в MAX как аудио-**файл** (mp3): отправить «нативное»
  голосовое библиотека не умеет.
- **Стикеры** приходят в MAX картинкой (MAX рисует её на светлой подложке —
  отправить настоящий стикер userbot не может). Анимированные — статичным кадром.

## 🧩 Как это работает

- **MAX-сторона** — userbot ([`maxapi-python`](https://pypi.org/project/maxapi-python/),
  внутренний WebSocket-протокол), слушает события аккаунта.
- **Telegram-сторона** — обычный бот ([`aiogram`](https://docs.aiogram.dev/) 3) в
  супергруппе-форуме; темы = forum topics.
- **Медиа** — скачиваются и переотправляются; конвертация стикеров/аудио через
  [`Pillow`](https://python-pillow.org/) и `ffmpeg`.
- **Связи чат↔тема и позиция** — в SQLite.

```
bot.py          точка входа: userbot MAX + бот Telegram одновременно
login.py        одноразовая авторизация MAX (SMS-код)
get_group_id.py утилита: узнать ID группы
max_to_tg.py    MAX → Telegram
tg_to_max.py    Telegram → MAX
config.py       настройки из .env
storage.py      SQLite: связи MAX-чат ↔ Telegram-тема
install.sh      установщик в одну команду
```

## 🩺 Траблшутинг

- **Темы не создаются** (`not enough rights to create a topic`) — бот не админ
  группы или нет права «Управление темами»; либо в группе не включены Темы.
- **`login.py` зависает под systemd** — сервис не интерактивный; сначала пройди
  авторизацию вручную (`login.py`), потом запускай сервис.
- **Бот не видит сообщения в Telegram** — не отключён Privacy Mode (`/setprivacy`).
- **Из Telegram не уходит в MAX** — пиши **в теме**, созданной мостом; «General» и
  ручные несвязанные темы игнорируются. Проверь `FORWARD_TG_TO_MAX=true`.
- **Аккаунт MAX «разлогинило»** — удали `max_session/` и пройди `login.py` заново.
- **Форвард/звонок не дошёл** — поставь `LOG_LEVEL=DEBUG`, повтори, посмотри лог.
